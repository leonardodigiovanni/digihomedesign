# Marketing — registro campagne con media e testo

**Data:** 2026-04-11  
**Stato:** completato ✓ (inclusi fix upload e cancellazione file)

---

## Obiettivo

Sezione `/marketing` dove ogni record rappresenta un'azione di marketing con:
- **Tipo** (es. "Volantinaggio", "Social", "Email", "Evento", …) — campo testo libero o select da lista
- **Titolo** descrittivo
- **Immagine** — upload opzionale (jpg/png/webp/gif)
- **Video** — upload opzionale (mp4/mov/webm)
- **Periodo** — campo testo libero (es. "Giugno 2025", "Q1 2026") o range date
- **Note** — testo libero

---

## Struttura DB — tabella `marketing`

| Colonna     | Tipo                  | Note                              |
|-------------|-----------------------|-----------------------------------|
| `id`        | INT PK AUTO           |                                   |
| `tipo`      | VARCHAR(100) NOT NULL | Es. "Volantinaggio", "Social"     |
| `titolo`    | VARCHAR(200) NOT NULL |                                   |
| `periodo`   | VARCHAR(100) NOT NULL DEFAULT '' | Testo libero                |
| `immagine`  | VARCHAR(255) NULL     | Filename in `/uploads/marketing/` |
| `video`     | VARCHAR(255) NULL     | Filename in `/uploads/marketing/` |
| `note`      | TEXT NULL             |                                   |
| `created_at`| TIMESTAMP DEFAULT CURRENT_TIMESTAMP |                    |

---

## Upload file

Riuso della stessa logica di `/api/upload-cantiere`:
- Nuova API route `/api/upload-marketing` che salva in `public/uploads/marketing/`
- La form carica immagine e/o video separatamente con due input file

---

## UI

- Tabella record: Tipo, Titolo, Periodo, Immagine (anteprima thumbnail), Video (icona), Note (troncata), Azioni
- Filtro per tipo + ricerca testo
- Ordinamento: created_at, tipo, periodo
- Pulsante "+ Nuovo record" → form espandibile:
  - Tipo (select con opzioni predefinite + voce "Altro")
  - Titolo
  - Periodo (testo)
  - Note (textarea)
  - Upload immagine (opzionale)
  - Upload video (opzionale)
- Per ogni riga: pulsante Apri (modal dettaglio con media a grandezza), pulsante Elimina

### Tipi predefiniti
`Volantinaggio`, `Social Media`, `Email`, `Evento`, `Sponsorizzazione`, `Affissione`, `Fiera`, `Altro`

## Fix post-implementazione

### Bug: upload corrompeva i file binari
`file.arrayBuffer()` in Next.js App Router tronca i file di medie dimensioni (~86KB persi su 1MB).  
**Fix:** lettura tramite `file.stream().getReader()` con concatenazione dei chunk.

### Bug: immagine nel modal deformata
Il modal era renderizzato inline dentro `<tbody>` (sibling di `<tr>` in RecordRow).  
**Fix:** `createPortal(jsx, document.body)` — modal reso fuori dall'albero della tabella.  
Il `DettaglioModal` usa `onLoad` per leggere le dimensioni naturali e imposta `aspectRatio` esplicito.

### Feature: cancellazione file fisici
`deleteRecord` ora legge immagine/video dal DB prima della DELETE e rimuove i file da `public/uploads/marketing/` tramite `fs/promises unlink`.
