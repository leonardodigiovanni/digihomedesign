# Fatture — Fatture Attive e Passive

**Data:** 2026-04-10  
**Stato:** completato

---

## Obiettivo

Implementare la pagina `/fatture` con due sezioni distinte:
- **Fatture Attive** — fatture emesse (clienti che ci devono pagare)
- **Fatture Passive** — fatture ricevute (fornitori che dobbiamo pagare)

Ogni sezione ha una tabella con colonne filtrabili e ordinabili lato client.

---

## Struttura DB

Tabella unica `fatture` creata automaticamente se non esiste:

| Colonna        | Tipo           | Note                         |
|----------------|----------------|------------------------------|
| `id`           | INT PK AUTO    |                              |
| `tipo`         | ENUM('attiva','passiva') |                  |
| `numero`       | VARCHAR(50)    | Numero fattura               |
| `data`         | DATE           |                              |
| `controparte`  | VARCHAR(200)   | Cliente o Fornitore          |
| `importo`      | DECIMAL(10,2)  |                              |
| `iva`          | DECIMAL(5,2)   | Percentuale IVA              |
| `totale`       | DECIMAL(10,2)  | Importo + IVA                |
| `pagata`       | TINYINT(1)     | 0=no, 1=sì                   |
| `note`         | TEXT           | Facoltativo                  |
| `created_at`   | TIMESTAMP      |                              |

---

## Colonne tabella

Entrambe le sezioni mostrano: **Numero**, **Data**, **Controparte**, **Importo**, **IVA %**, **Totale**, **Pagata**, **Note**, **Azioni** (elimina).

Ogni colonna ha:
- **Ordinamento** — click sull'header alterna ASC/DESC
- **Filtro** — input testo sotto l'header (per colonne stringa) o select (per Pagata)

---

## Funzionalità

- Aggiunta fattura tramite form (tipo, numero, data, controparte, importo, IVA%, pagata, note)
- Eliminazione fattura
- Toggle pagata/non pagata direttamente dalla tabella
- Totali riepilogo per sezione: totale imponibile, totale IVA, totale generale, da pagare

---

## File coinvolti

| File | Azione |
|---|---|
| `app/fatture/page.tsx` | Server component: guard accesso, fetch fatture, passa a client |
| `app/fatture/fatture-client.tsx` | Client component: UI con le due sezioni, filtri, ordinamento |

---

## Scelte tecniche

- Filtri e ordinamento: gestiti interamente lato client (nessuna query al DB per ogni filtro)
- Stili: inline come da convenzione del progetto
- Bottoni: classi `btn-green`, `btn-red`, `btn-gray`
