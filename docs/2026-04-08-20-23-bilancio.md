# Bilancio — Conto Economico + Stato Patrimoniale

**Data:** 2026-04-08  
**Stato:** completato (abbozzo v1)

---

## Obiettivo

Implementare la pagina `/bilancio` con due sezioni principali:

1. **Conto Economico** — ricavi e costi filtrabili per anno, con calcolo automatico del risultato d'esercizio.
2. **Stato Patrimoniale** — snapshot di attivo e passivo, aggiornato in tempo reale ad ogni movimento contabile.

Ogni volta che si registra un'**entrata** o **uscita**, entrambe le sezioni si aggiornano automaticamente.

---

## Database

### Nuova tabella: `movimenti_contabili`

```sql
CREATE TABLE IF NOT EXISTS movimenti_contabili (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  data          DATE NOT NULL,
  anno          INT NOT NULL,                        -- anno estratto da data (per filtro)
  tipo          ENUM('entrata','uscita') NOT NULL,
  sezione_ce    VARCHAR(100) NOT NULL,               -- categoria conto economico (es. 'Ricavi vendite', 'Costi materie prime')
  sezione_sp    VARCHAR(100) NOT NULL,               -- categoria stato patrimoniale (es. 'Disponibilità liquide', 'Debiti')
  descrizione   TEXT,
  importo       DECIMAL(10,2) NOT NULL,
  created_by    VARCHAR(100) NOT NULL DEFAULT '',
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

L'anno viene salvato esplicitamente per semplicità di query (evita YEAR(data) nelle GROUP BY).

## Categorie predefinite

### Conto Economico — `sezione_ce`

**Entrate (ricavi):**
- `Ricavi delle vendite`
- `Prestazioni di servizi`
- `Altri ricavi`
- `Proventi finanziari`

**Uscite (costi):**
- `Materie prime e materiali`
- `Servizi`
- `Personale`
- `Ammortamenti`
- `Oneri finanziari`
- `Oneri diversi`
- `Imposte`

### Stato Patrimoniale — `sezione_sp`

**Attivo:**
- `Disponibilità liquide` ← entrate cash vanno qui
- `Crediti`
- `Rimanenze`
- `Immobilizzazioni`

**Passivo:**
- `Debiti` ← uscite non ancora pagate vanno qui
- `Fondi e TFR`
- `Capitale sociale`
- `Utile/Perdita d'esercizio` ← calcolato dinamicamente da CE

---

## Logica di aggiornamento

- **Conto Economico**: somma `importo` per anno e `sezione_ce`. Risultato = totale entrate − totale uscite.
- **Stato Patrimoniale attivo**: somma entrate per `sezione_sp`. Voce "Utile/Perdita" riflette il risultato CE dell'anno corrente.
- **Stato Patrimoniale passivo**: somma uscite per `sezione_sp`.
- Nessun SSE per ora — aggiornamento via revalidazione server action (server component ricaricato dopo ogni salvataggio).

## Note tecniche

- Il form ha due `<select>` per `sezione_ce` e `sezione_sp` con le categorie predefinite.
- Il filtro anno è un `<select>` client-side che ricarica i dati tramite una query server-side (tramite `router.push` con searchParam `?anno=2026` oppure revalidation).
- Per la prima versione nessuna modifica alle fatture esistenti — i movimenti sono indipendenti.
- Il magazzino (giacenze) potrebbe alimentare automaticamente "Rimanenze" in SP in futuro, ma non ora.
