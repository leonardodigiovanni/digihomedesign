# Magazzino — sezione dipendenti

**Data:** 2026-04-02  
**Stato:** completato

---

## Obiettivo

Implementare la sezione Magazzino (pagina 16) completamente funzionante per i ruoli `dipendente` e superiori. Permette di visualizzare, filtrare, ordinare e aggiungere materiali a magazzino.

---

## Struttura dati — tabella MySQL `magazzino`

```sql
CREATE TABLE magazzino (
  id              INT AUTO_INCREMENT PRIMARY KEY,
  descrizione     VARCHAR(255) NOT NULL,
  produttore      VARCHAR(100) NOT NULL,
  modello         VARCHAR(100) NOT NULL,
  costo_unitario  DECIMAL(10,2) NOT NULL,
  tipo_unita      VARCHAR(50)  NOT NULL,   -- es. pz, m², kg, ml…
  giacenza        DECIMAL(10,2) NOT NULL DEFAULT 0,
  totale_caricato DECIMAL(10,2) NOT NULL DEFAULT 0,
  created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## Funzionalità

### Griglia (stile gestione-utenti)
- Colonne: **Descrizione**, **Produttore**, **Modello**, **Costo/unità**, **Tipo unità**, **Giacenza**, **Totale caricato**, **Aggiunto il**
- Ordinamento cliccabile su tutte le colonne
- Filtro testo (cerca su descrizione, produttore, modello)
- Filtro tipo unità (select)

### Form "Aggiungi materiale"
- Bottone **+ Aggiungi materiale** in alto a destra
- Apre un modal/form inline con i campi:
  - Descrizione (text, obbligatorio)
  - Produttore (text, obbligatorio)
  - Modello (text, obbligatorio)
  - Costo per unità (number, decimale, obbligatorio)
  - Tipo di unità (text con suggerimenti: pz, m², kg, ml, mt, lt)
  - Giacenza iniziale (number, decimale, default 0)
- Al submit: `addMateriale` server action → INSERT in DB → `router.refresh()`
- `totale_caricato` viene inizializzato uguale alla giacenza iniziale al momento dell'inserimento

### Azioni per riga (future — non in questo sprint)

## Scelte tecniche

- Pattern identico a `gestione-utenti`: server component fetcha i dati, passa array al client component
- `useActionState` per il form di inserimento
- Modal implementato con stato locale (`useState`) senza librerie esterne, overlay scuro
- Inline styles come nel resto del progetto
