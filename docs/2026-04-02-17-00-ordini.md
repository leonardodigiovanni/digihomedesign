# Ordini — ristrutturazione completa

**Data:** 2026-04-02  
**Stato:** in pianificazione

---

## Obiettivo

Ristrutturare la gestione ordini in tre sezioni distinte:

1. **Ordini a Fornitori** (ID 26, rinominato) — ordini inviati ai fornitori
2. **Ordini Ricevuti** (ID 35, nuovo) — ordini ricevuti dai clienti, con pannello Note interno
3. **I Miei Ordini** (ID 30, esistente) — vista "Ordini Ricevuti" filtrata per il cliente loggato

---

## Struttura dati

### Tabella `ordini_fornitori`
```sql
CREATE TABLE IF NOT EXISTS ordini_fornitori (
  id             INT AUTO_INCREMENT PRIMARY KEY,
  numero_ordine  VARCHAR(50)   NOT NULL DEFAULT '',
  fornitore      VARCHAR(255)  NOT NULL,
  descrizione    TEXT          NOT NULL DEFAULT '',
  stato          ENUM('bozza','inviato','confermato','ricevuto','annullato') NOT NULL DEFAULT 'bozza',
  totale         DECIMAL(10,2) NOT NULL DEFAULT 0,
  data_ordine    DATE          NOT NULL,
  created_by     VARCHAR(100)  NOT NULL DEFAULT '',
  created_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Tabella `ordini_ricevuti`
```sql
CREATE TABLE IF NOT EXISTS ordini_ricevuti (
  id             INT AUTO_INCREMENT PRIMARY KEY,
  numero_ordine  VARCHAR(50)   NOT NULL DEFAULT '',
  cliente        VARCHAR(100)  NOT NULL,   -- username del cliente
  descrizione    TEXT          NOT NULL DEFAULT '',
  stato          ENUM('nuovo','in_lavorazione','completato','annullato') NOT NULL DEFAULT 'nuovo',
  totale         DECIMAL(10,2) NOT NULL DEFAULT 0,
  data_ordine    DATE          NOT NULL,
  created_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Tabella `ordini_note` (solo staff interno)
```sql
CREATE TABLE IF NOT EXISTS ordini_note (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  ordine_id  INT          NOT NULL,
  testo      TEXT         NOT NULL,
  autore     VARCHAR(100) NOT NULL,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (ordine_id) REFERENCES ordini_ricevuti(id) ON DELETE CASCADE
);
```

## Funzionalità

### Ordini a Fornitori
- Griglia con colonne: Numero, Fornitore, Descrizione, Stato, Totale, Data, Azioni
- Filtro testo + filtro stato
- Ordinamento su tutte le colonne
- Modal "Nuovo ordine" — tutti i campi
- Cambio stato inline per riga
- Elimina (solo admin)

### Ordini Ricevuti (staff)
- Griglia identica con colonne: Numero, Cliente, Descrizione, Stato, Totale, Data
- Riga espandibile (clic) → mostra **pannello Note** con:
  - Lista note con autore e timestamp
  - Form aggiungi nota (textarea + tasto Aggiungi)
- Cambio stato inline
- Elimina (solo admin)

### I Miei Ordini (cliente)
- Stessa griglia di Ordini Ricevuti filtrata su `cliente = session_user`
- **Pannello Note assente** — il cliente non vede le note interne
- No cambio stato (sola lettura per il cliente)
- Visibile ai ruoli `cliente` (e superiori, tramite matrice permessi)

---

## Logica Note
- Il pannello Note è renderizzato solo se `role !== 'cliente'`
- Le note non vengono mai inviate al client "I Miei Ordini"
- La server action `addNota` verifica che `role` non sia `cliente`
