# Cantieri — sezione dedicata con accesso differenziato per ruolo

**Data:** 2026-04-10  
**Stato:** completato

---

## Obiettivo

Pagina `/cantieri` che mostra:
- **Cliente loggato** → solo i propri cantieri, con campi economici nascosti
- **Dipendente / Admin** → tutti i cantieri, tutti i campi visibili

Ogni cantiere ha: anagrafica cliente, date, stato, elenco lavori (con prezzi/sconti), raccolta foto/video, note interne.

---

## Struttura DB

### `cantieri`
| Colonna        | Tipo                                                       | Note                          |
|----------------|------------------------------------------------------------|-------------------------------|
| `id`           | INT PK AUTO                                               |                               |
| `cliente_id`   | INT FK → anagrafica_clienti.id                            |                               |
| `titolo`       | VARCHAR(200)                                              | Nome/descrizione del cantiere |
| `indirizzo`    | VARCHAR(300)                                              | Luogo dei lavori              |
| `stato`        | ENUM('preventivo','in_corso','completato','sospeso')      |                               |
| `inizio_lavori`| DATE NULL                                                  |                               |
| `fine_lavori`  | DATE NULL                                                  |                               |
| `note_pubbliche`| TEXT NULL                                                 | Visibili al cliente           |
| `note_interne` | TEXT NULL                                                  | Solo staff                    |
| `created_at`   | TIMESTAMP                                                  |                               |

### `cantieri_lavori`
| Colonna        | Tipo           | Note                          |
|----------------|----------------|-------------------------------|
| `id`           | INT PK AUTO   |                               |
| `cantiere_id`  | INT FK         |                               |
| `descrizione`  | VARCHAR(300)   |                               |
| `qta`          | DECIMAL(10,2)  |                               |
| `unita`        | VARCHAR(20)    | Es. "mq", "ml", "cad"        |
| `prezzo_unit`  | DECIMAL(10,2)  | Solo staff                    |
| `sconto_pct`   | DECIMAL(5,2)   | Solo staff                    |
| `totale`       | DECIMAL(10,2) GENERATED | imponibile riga     |
| `visibile_cliente` | TINYINT(1) DEFAULT 1 | Se mostrare al cliente |

### `cantieri_media`
| Colonna        | Tipo                    | Note                          |
|----------------|-------------------------|-------------------------------|
| `id`           | INT PK AUTO            |                               |
| `cantiere_id`  | INT FK                  |                               |
| `tipo`         | ENUM('foto','video')    |                               |
| `filename`     | VARCHAR(255)            | Nome file su disco            |
| `descrizione`  | VARCHAR(200) NULL       |                               |
| `visibile_cliente` | TINYINT(1) DEFAULT 1|                               |
| `created_at`   | TIMESTAMP               |                               |

## Navigazione

- Route: `app/cantieri/page.tsx` (unica route, contenuto differenziato per ruolo)
- `lib/nav-config.ts`: aggiornare id 28 (Cantieri staff) a `/cantieri` e id 31 (I Miei Cantieri) a `/cantieri`
- Il server component legge il ruolo e il username dal cookie, passa entrambi al client

---

## UI per ruolo

### Cliente
- Lista dei propri cantieri con: titolo, indirizzo, stato (badge), date
- Per ogni cantiere: note pubbliche, lavori visibili (senza prezzi), foto/video visibili

### Staff / Admin
- Lista **tutti** i cantieri con colonna cliente
- Per ogni cantiere: tutti i campi inclusi prezzi, sconti, note interne
- Pulsanti: aggiungi cantiere, aggiungi lavoro, aggiungi media, elimina

---

## Upload media

- Form con `<input type="file">` che carica verso `/api/upload-cantiere`
- File salvati in `public/uploads/cantieri/{cantiere_id}/`
- Servi le immagini via `<img src="/uploads/cantieri/{id}/{filename}">` o video con `<video>`
