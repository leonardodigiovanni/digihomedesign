# Worklist — compiti assegnati ai dipendenti

**Data:** 2026-04-11  
**Stato:** completato

---

## Obiettivo

Sezione `/worklist` dove:
- **Direttore / Admin**: vede tutti i compiti, crea nuovi compiti e li assegna a qualsiasi dipendente (o se stesso), può eliminare
- **Dipendenti / altri ruoli con accesso**: vedono solo i compiti assegnati a loro, possono marcare come eseguiti ma non creare né eliminare

---

## Ruoli e accesso

| Azione                  | Admin / Direttore | Dipendente / altri |
|-------------------------|-------------------|--------------------|
| Vede tutti i compiti    | ✓                 | ✗                  |
| Vede i propri compiti   | ✓                 | ✓                  |
| Crea compito            | ✓                 | ✗                  |
| Assegna a utente        | ✓                 | ✗                  |
| Marca come eseguito     | ✓                 | ✓ (solo i propri)  |
| Elimina                 | ✓                 | ✗                  |

I "gestori" (chi può creare) sono: `admin`, `direttore`.  
Chi può accedere alla worklist è chiunque abbia l'id 27 nei propri rolePermissions (configurabile in Settings).

---

## Struttura DB — tabella `worklist`

| Colonna       | Tipo                                              | Note                        |
|---------------|---------------------------------------------------|-----------------------------|
| `id`          | INT PK AUTO                                       |                             |
| `titolo`      | VARCHAR(200) NOT NULL                             |                             |
| `descrizione` | TEXT NULL                                         |                             |
| `assegnato_a` | VARCHAR(100) NOT NULL                             | username del dipendente     |
| `creato_da`   | VARCHAR(100) NOT NULL                             | username del gestore        |
| `priorita`    | ENUM('bassa','normale','alta','urgente') DEFAULT 'normale' |                  |
| `stato`       | ENUM('da_fare','in_corso','completato') DEFAULT 'da_fare' |                   |
| `data_scadenza` | DATE NULL                                       |                             |
| `created_at`  | TIMESTAMP DEFAULT CURRENT_TIMESTAMP               |                             |

## UI

### Vista gestore (admin / direttore)

- Tabella con tutti i compiti: Titolo, Assegnato a, Priorità, Stato, Scadenza, Creato da, Azioni
- Filtri: stato, assegnato_a, priorità
- Pulsante "+ Nuovo compito" → form inline o modal: titolo*, descrizione, assegnato_a (select utenti con accesso worklist), priorità, data_scadenza
- Per ogni riga: pulsante Elimina (solo admin/direttore) + select stato

### Vista dipendente

- Tabella con solo i propri compiti: Titolo, Priorità, Stato, Scadenza, Creato da
- Filtro stato
- Per ogni riga: pulsante/select per cambiare stato (da_fare → in_corso → completato)
- Nessun pulsante crea/elimina

### Badge priorità

| Valore  | Colore           |
|---------|------------------|
| bassa   | grigio `#888`    |
| normale | blu `#1565c0`    |
| alta    | arancio `#e65100`|
| urgente | rosso `#c62828`  |

### Badge stato

| Valore     | Colore           |
|------------|------------------|
| da_fare    | grigio `#757575` |
| in_corso   | arancio `#e65100`|
| completato | verde `#2e7d32`  |
